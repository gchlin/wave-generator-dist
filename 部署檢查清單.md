# GitHub Pages 部署檢查清單

## 📋 開始前準備

- [ ] 已有 GitHub 帳號並登入
- [ ] 已下載並解壓縮 `wave-generator-dist` 資料夾
- [ ] 確認資料夾內有 `index.html` 和 `assets` 資料夾

---

## ✅ 步驟 1：建立 Repository

- [ ] 點選 GitHub 右上角 `+` → `New repository`
- [ ] Repository name 填寫：`wave-generator`
- [ ] 選擇 **Public**（公開）
- [ ] **不要勾選** "Add a README file"
- [ ] **不要勾選** "Add .gitignore"
- [ ] **不要勾選** "Choose a license"
- [ ] 點選 `Create repository`
- [ ] 看到空白的 repository 頁面

---

## ✅ 步驟 2：上傳檔案

- [ ] 點選 "uploading an existing file" 連結
- [ ] 打開 `wave-generator-dist` 資料夾（在電腦上）
- [ ] **全選資料夾內的所有檔案**（Ctrl+A / Cmd+A）
- [ ] 確認選中的是檔案，不是資料夾本身
- [ ] 拖拽檔案到 GitHub 上傳區
- [ ] 等待上傳完成
- [ ] 看到檔案列表：`index.html`, `assets/`, 等
- [ ] Commit message 輸入：`Upload website files`
- [ ] 點選 `Commit changes`

### 上傳後檢查

- [ ] repository 根目錄有 `index.html`
- [ ] repository 根目錄有 `assets` 資料夾
- [ ] assets 資料夾內有 `.css` 和 `.js` 檔案
- [ ] **沒有** `wave-generator-dist` 這層資料夾

---

## ✅ 步驟 3：啟用 GitHub Pages

- [ ] 點選 repository 頁面的 `Settings`
- [ ] 在左側選單找到 `Pages`
- [ ] Source 選擇：`Deploy from a branch`
- [ ] Branch 選擇：`main`（或 `master`）
- [ ] Folder 選擇：`/ (root)`
- [ ] 點選 `Save`
- [ ] 等待 1-2 分鐘
- [ ] 重新整理頁面
- [ ] 看到綠色訊息：`Your site is live at https://...`

---

## ✅ 步驟 4：測試網站

- [ ] 複製網址：`https://你的使用者名稱.github.io/wave-generator/`
- [ ] 在瀏覽器開啟網址
- [ ] 網站能正常顯示
- [ ] 標題顯示：「波動圖形產生器」
- [ ] 左側有參數控制面板
- [ ] 中間有波形預覽區
- [ ] 可以調整滑桿（振幅、波長、相位）
- [ ] 波形會即時變化
- [ ] 可以點選「下載 SVG」
- [ ] 可以切換配色方案
- [ ] 可以點選快速預設場景

---

## 🔍 檔案結構檢查

打開你的 repository，應該看到：

```
✅ index.html                    (在根目錄)
✅ assets/
   ✅ index-[一串英數字].css
   ✅ index-[一串英數字].js
✅ vite.svg                      (可能有)

❌ wave-generator-dist/          (不應該看到這個)
❌ node_modules/                 (不應該看到這個)
❌ src/                          (不應該看到這個)
```

## ❓ 常見問題自我診斷

### 問題：顯示 404 Not Found

檢查：
- [ ] `index.html` 是否在根目錄？
- [ ] 網址是否正確？
- [ ] GitHub Pages 是否已啟用？
- [ ] 是否等待了 2-3 分鐘？

解決：重新檢查檔案結構，確保 `index.html` 在根目錄

---

### 問題：網頁樣式跑掉

檢查：
- [ ] `assets` 資料夾是否存在？
- [ ] assets 內是否有 .css 和 .js 檔案？
- [ ] 檔案是否都上傳成功？

解決：重新上傳 `assets` 資料夾

---

### 問題：找不到 Pages 設定

檢查：
- [ ] Repository 是否為 Public？
- [ ] 是否在正確的 repository？

解決：Settings → General → Change visibility → Public
